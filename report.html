### Using a LeNet architecture to begin with
import tensorflow as tf
from tensorflow.contrib.layers import flatten

EPOCHS = 20
BATCH_SIZE = 128

def LeNet(x):    
    # Hyperparameters
    mu = 0
    sigma = 0.08
    
    # Layer 1: Convolutional. Input = 32x32x3. Output = 28x28x6.
    conv1_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 3, 6), mean = mu, stddev = sigma))
    conv1_b = tf.Variable(tf.zeros(6))
    conv1   = tf.nn.conv2d(x, conv1_W, strides=[1, 1, 1, 1], padding='VALID') + conv1_b

    # Activation.
    conv1 = tf.nn.relu(conv1)

    # Pooling. Input = 28x28x6. Output = 14x14x6.
    conv1 = tf.nn.max_pool(conv1, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')

    # Layer 2: Convolutional. Output = 10x10x16.
    conv2_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 6, 16), mean = mu, stddev = sigma))
    conv2_b = tf.Variable(tf.zeros(16))
    conv2   = tf.nn.conv2d(conv1, conv2_W, strides=[1, 1, 1, 1], padding='VALID') + conv2_b
    
    # Activation.
    conv2 = tf.nn.relu(conv2)

    # Pooling. Input = 10x10x16. Output = 5x5x16.
    conv2 = tf.nn.max_pool(conv2, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')

    # Flatten. Input = 5x5x16. Output = 400.
    fc0   = flatten(conv2)
    
    # Layer 3: Fully Connected. Input = 400. Output = 120.
    fc1_W = tf.Variable(tf.truncated_normal(shape=(400, 120), mean = mu, stddev = sigma))
    fc1_b = tf.Variable(tf.zeros(120))
    fc1   = tf.matmul(fc0, fc1_W) + fc1_b
    
    # Activation.
    fc1    = tf.nn.relu(fc1)

    # Layer 4: Fully Connected. Input = 120. Output = 84.
    fc2_W  = tf.Variable(tf.truncated_normal(shape=(120, 84), mean = mu, stddev = sigma))
    fc2_b  = tf.Variable(tf.zeros(84))
    fc2    = tf.matmul(fc1, fc2_W) + fc2_b
    
    # Activation.
    fc2    = tf.nn.relu(fc2)
    
    # Layer 5: Fully Connected. Input = 84. Output = 43.
    fc3_W  = tf.Variable(tf.truncated_normal(shape=(84, 43), mean = mu, stddev = sigma))
    fc3_b  = tf.Variable(tf.zeros(43))
    logits = tf.matmul(fc2, fc3_W) + fc3_b
    
    return logits

Question 3¶

What does your final architecture look like? (Type of model, layers, sizes, connectivity, etc.) For reference on how to build a deep neural network using TensorFlow, see Deep Neural Network in TensorFlow from the classroom.

Answer: The architecture is modeled off of LeNet - i.e, 1 convnet, 1 relu, 1 max-pool, 1 convnet again, 1 relu, 1 max-pool, flatten, 1 FC layer, 1 relu, 1 FC layer again.

The sizes were as follows 1 convnet converts 32x32x3 image to 28x28x6 with filter size 5x5 and 6 filters used.

The max-pool converts the outputs to 14x14x6 with a stride of (2,2) over the horizontal and vertical regions of the image.

The next convnet converts 14x14x6 to 10x10x16. Max-pool converts it to 5x5x16.

The Fully-Connected layer then takes the flattened data as input (5x5x16=400) and outputs single dimensional length of 120.

After applying a relu and through the next FC, the output is at 84.The final output to 43 is obtained as the final set of weights and bias are applied.

### Model Evaluation
correct_prediction = tf.equal(tf.argmax(logits, 1), tf.argmax(one_hot_y, 1))
accuracy_operation = tf.reduce_mean(tf.cast(correct_prediction, tf.float32))
saver = tf.train.Saver()

def evaluate(X_data, y_data):
    num_examples = len(X_data)
    total_accuracy = 0
    sess = tf.get_default_session()
    for offset in range(0, num_examples, BATCH_SIZE):
        batch_x, batch_y = X_data[offset:offset+BATCH_SIZE], y_data[offset:offset+BATCH_SIZE]
        accuracy = sess.run(accuracy_operation, feed_dict={x: batch_x, y: batch_y})
        total_accuracy += (accuracy * len(batch_x))
    return total_accuracy / num_examples

Question 5¶

What approach did you take in coming up with a solution to this problem? It may have been a process of trial and error, in which case, outline the steps you took to get to the final solution and why you chose those steps. Perhaps your solution involved an already well known implementation or architecture. In this case, discuss why you think this is suitable for the current problem.

Answer: I tried starting off with the LeNet as a baseline architecture along with some hyperparamters used there as defaults. With this I would end up with a validation accuracy of 95%. So I started playing around with the architecture itself, among other paramters, to improve the validation accuracy.

  1. Changing Epoch from 10 to 20 seems to give good results. I started off right from the first training epoch, at a high validation acccuracy of around 72%, eventually ending up with 96.5% as against starting off with 53% and ending up with 95%.

  2. I then changed the batch size to 256 from 128 (keeping epochs at 20), but that didn't seem to help too much as the accuracy actually gave me just about the same (around 96%). It perhaps isn't worthwhile to increase the memory footprint for almost negligible gain.

  3. Changing some of the hyper parameters such as the stddev from 0.1 to 0.2, at 20 epochs I lost my accuracy and it fell down to below 10%. Perhaps it has to be closer to the mean, so trying a value of 0.08 with 20 epochs seems to be giving around 97.8% accuracy which seems better!

  4. Changing the convolutional layer filter size, to 7x7 on the first 2 conv layers, required eliminating max pooling on the second conv layer while adding an extra FC layer to downsample to 43 outputs in the end. The model gave about 97% accuracy, about 0.8 lower than the earlier 5x5 filter size. Changing it to a lower filter size of 3x3 also does not improve the validation accuracy. It came down to 95.5%.

  5. Playing around with learning rate didn't help much either. I tried increasing the learning rate to 0.005, which resulted in a very low validation accuracy of 5%. Decreasing it to 0.0005 didn't seem to improve the validation accuracy or speed much either. The accuracy got to around 96.5% which was marginally close to what I got with 0.001. It seems like the optimal learning rate around 0.001 is just right for the Adam Optimizer.
